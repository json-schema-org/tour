# $schema 
In JSON Schema, `$schema` is a **string** that defines the version of the JSON Schema standard that the [schema](https://json-schema.org/learn/glossary#schema) is written in. 

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "string"
}
```
In the above example, the schema is written in the JSON Schema [draft](https://json-schema.org/learn/glossary#draft) 2020-12. you can find the latest version of JSON Schema [here](https://json-schema.org/specification-links).

For the tour, we have used the JSON Schema draft 2020-12.

# $id 

The `$id` [keyword](https://json-schema.org/learn/glossary#keyword) is a **string** that defines the URI of the schema. 

```json
{
  "$id": "https://example.com/person",
  "type": "object",
  "properties": {
    "name": {
      "type": "string"
    }
  }
}
```

In the above example, the schema is given a URI of `https://example.com/person`.

The schema URI can be used to reference the schema from other schemas using the `$ref` keyword.
```json
{
  "$ref": "https://example.com/person"
}
```
